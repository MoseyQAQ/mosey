## 课题组内自建超算的每卡时成本计算公式

`$C_{卡时}=\frac{C_{总建}+t_{折旧}\times\left(C_{月人}+C_{月电}\right)}{720\times t_{折旧}\times N_\text{GPU}\times p_{用}\times f_{效}},$`

`$C_{月电} = \begin{cases} P_{峰} \times C_{月托}, & (\text{IDC托管模式}) \\ 720 \times C_{度电} \times (P_{空} + p_{用} \times (P_{典} - P_{空})). & (\text{电费直付模式}) \end{cases}$`


| `$C_$` | 每卡时核算成本 |
| `$C_$` | 集群（+机房）总建设成本（一次性） |
| `$C_$` | 每月人员成本 |
| `$C_$` | 每月电力成本（分为[IDC托管]和[电费直付]两种模式） |
| `$C_$` | 每月每kW托管成本（[IDC托管]模式，单位：元/kW·月） |
| `$C_$` | 每kWh电费成本（[电费直付]模式，单位：元/kWh） |
| `$P_$` | 超算满载峰值功率（[IDC托管]模式，单位：kW） |
| `$P_$` | 超算空载典型功率，考虑机房PUE（[电费直付]模式，单位：kW） |
| `$P_$` | 超算满载典型功率，考虑机房PUE和应用负载特性（[电费直付]模式，单位：kW） |
| `$p_$` | 超算在折旧周期内综合使用率 |
| `$t_$` | 超算折旧周期（单位：月） |
| `$N_\text$` | 超算计算节点GPU总数 |
| `$f_$` | 自建超算相较于SAI基线的效率因子（由于自建超算优化不到位） |


## 🌰举例

- 某课题组购入SAI SlimPOD-144超算（改进型V100-32G），`$C_{总建}$`= 150万元，`$C_{月人}$`= 10000 元/月。托管于学校IDC机房，`$C_{月托}$`= 425 元/kW·月（原型机LiuLab-HPC在校内超算机房的托管费用），`$P_{峰}$`= 60 kW（36台计算节点1.6kW/台，主节点和交换机等设备约2.4kW），`$p_{用}$`= 0.5（此使用率对于课题组超算来说已较高），`$t_{折旧}$`= 36月。此外由于招聘的专职运维工程师水平不如SAI团队，若与SAI对比，需除以效率因子`$f_{效}$`= 0.95。则每卡时核算成本

`$C_{卡时}=\frac{1500000+36\times\left(10000+60\times425\right)}{720\times36\times144\times0.5\times0.95}

=1.5669\ (元)$`

- 若此课题组不为超算配置专职运维工程师，由学生运维超算，而作为代价，此课题组超算**没有性能调优和调度优化**，若与SAI对比，需除以效率因子`$f_{效}$`= 0.8，则每卡时核算成本

`$C_{卡时}=\frac{1500000+36\times\left(60\times425\right)}{720\times36\times144\times0.5\times0.8}

=1.6196\ (元)$`

- 若此课题组进一步白嫖学校电费/托管费，而作为代价，学生使用积极性降低，超算使用率`$p_{用}$`= 0.4，则每卡时核算成本

`$C_{卡时}=\frac{1500000}{720\times36\times144\times0.4\times0.8}

=1.2559\ (元)$`

- 若此课题组采取自建机房方案（而非IDC托管），则考虑机房设施（机柜、空调、电缆、施工费等）在3年内的摊销成本，`$C_{总建}$`= 180万元，`$C_{度电}$`= 0.6 元/kWh，`$P_{空}$`= 15 kW，`$P_{典}$`= 55 kW，则上述3种情况的每卡时核算成本分别为

`$C_{卡时}=\frac{1800000+36\times\left(10000+720 \times0.6\times(15+0.5\times(55-15))\right)}{720\times36\times144\times0.5\times0.95}

=1.5253\ (元)$`

`$C_{卡时}=\frac{1800000+36\times720 \times0.6\times(15+0.5\times(55-15))}{720\times36\times144\times0.5\times0.8}

=1.5702\ (元)$`

`$C_{卡时}=\frac{1800000}{720\times36\times144\times0.4\times0.8}

=1.5070\ (元)$`

- 若考虑硬件故障等问题带来的额外时间浪费甚至经济支出，实际核算成本将会更高。
- 自建超算往往管理混乱（甚至往往不如友商超算平台），导致**学生科研的积极性降低**，这种损失难以计量！
- 作为参考，以预充值模式租用SAI，一次性充值200万元，预计优惠后每卡时价格**低于1元**。

## 📌结论

**租用SAI的性价比远高于自建超算**。

对于已有自建超算的用户，SAI支持与用户自建超算深度融合，实现算力按需调配，请看卫星集群和姊妹集群

## 💡 Q&A

Q：`$p_{用}$`为什么取0.5？

A：不可高看课题组超算的使用率，即使以原型机LiuLab-HPC的使用强度，超算在组员的主观感觉上常年满载、排长队，但严格统计得到的实际使用率也只有0.61，而使用强度能达到原型机水平的实验室少之又少。

我们需要考虑到假期、半夜等时间段，还要考虑到资源碎片化导致的“有空闲资源但排不上队”，实际上，周末和法定节假日都能接近满载的课题组集群并不存在，资源碎片化也是广泛存在的问题。

另一方面，根据开发者对亲自运维的多个不同课题组集群的观察统计，课题组内集群使用率普遍在0.3~0.6之间。

综上所述，`$p_{用}$`取0.5是合理的。
