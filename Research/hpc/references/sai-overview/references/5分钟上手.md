## 交互式命令行玩耍

我们可以通过任务提交和管理中提到的`srun`，直接进入计算节点玩耍。

比如用下面的命令，就可以在8V100V0分区调用8卡：
```bash {wrap}
srun --qos rush-8gpu --nodes=1 --ntasks=1 --gpus-per-node=8 --partition=8V100V0 --time=1 --pty bash
```

此时可以看到`@`后面的主机名从`login-01`变成了`8v100v0`前缀的计算节点，说明我们已经进入计算节点。

使用`nvidia-smi`命令，就能看到当前节点中的 8 张 V100-SXM2-32G（通过 NVLink 合并 256GB 显存）：


使用`nvidia-smi topo -m`命令可看到 GPU 与 100G IB 网卡、25G RoCE 网卡的互联拓扑，经典的DGX架构：


玩耍结束后，使用`exit`命令退出到登录节点（`login-01`），或直接关闭MobaXterm窗口，即可结束计费。

## 提交一个GMX计算任务

我们将基于SAI上预先准备好的输入文件开展一次GMX动力学模拟，逐行执行以下命令即可（可直接复制粘贴）：
```bash
cp -pr /home/public/tutorial/GMX ~
cd GMX
sbatch gpu_gmx-singlegpu.sbatch
squeue
ssh 4v100pxn01   # 此处4v100pxn01需要改成squeue输出的任务所在Nodelist
nvidia-smi
btop   # 按q退出btop界面
```
